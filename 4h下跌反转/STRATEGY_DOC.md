# 4H 下跌反转策略文档（多单专用）

## 1. 策略概述

**策略名称**: 4H Downtrend Reversal Long Strategy (Long Only)
**时间周期**: 4小时
**策略类型**: 下跌反转做多策略
**交易方向**: 仅做多
**交易理念**: 在多头趋势和空头趋势之间的转换点捕捉反转做多机会，利用EMA12与EMA144的交叉作为开仓信号
**默认仓位**: 账户权益的20%

## 2. 技术指标

### 2.1 EMA指标
策略使用3条EMA均线：
- **EMA12**: 12周期指数移动平均线（短期均线）
- **EMA144**: 144周期指数移动平均线（中期均线，关键）
- **EMA565**: 565周期指数移动平均线（长期均线）

### 2.2 趋势判断

**多头趋势信号**:
```
EMA12 > EMA144 > EMA565
```
三条均线按照多头顺序排列。

**空头趋势信号**:
```
EMA12 < EMA144 < EMA565
```
三条均线按照空头顺序排列。

## 3. 交易逻辑

### 3.1 三标记位状态管理

策略使用三个标记位来管理状态转换，确保信号的精确性：

#### 标记位1：`was_uptrend`（曾经是多头趋势）
- **初始状态**: `false`
- **触发条件**: 检测到多头趋势（`EMA12 > EMA144 > EMA565`）
- **动作**:
  - 设置 `was_uptrend = true`
  - 清空所有记录数据和计数器

#### 标记位2：`was_downtrend`（曾经是空头趋势）
- **初始状态**: `false`
- **触发条件**: 检测到空头趋势（`EMA12 < EMA144 < EMA565`）且 `was_uptrend = true`
- **动作**:
  - 设置 `was_downtrend = true`
  - 设置 `strategy_active = true`
  - 初始化记录变量

#### 标记位3：`strategy_active`（策略激活）
- **初始状态**: `false`
- **设置为true**: 从多头趋势转换为空头趋势时
- **设置为false**: 当策略激活状态下再次检测到多头趋势时
- **作用**: 控制是否记录EMA交叉事件和记录最低点

### 3.2 多单逻辑（下跌反转做多）

**策略思路**:
- 在多头和空头趋势之间的转换时启动策略
- 从多头转为空头时，开始记录关键最低点
- 记录EMA12上穿和下穿事件
- 第二次上穿后，在特定位置下限价单

**关键最低点记录**:

#### low1（空头趋势中EMA12的最低点）：
- 记录条件：`strategy_active = true` 且 `downtrend_signal = true`
- 记录对象：EMA12的最低值
- 用途：作为多单的止损位置

#### price_low_downtrend（空头趋势中的现价最低点）：
- 记录条件：`strategy_active = true` 且 `downtrend_signal = true`
- 记录对象：K线的最低价（`low`）
- 用途：作为止损参考

#### price_low_after_cu（下穿后到上穿之间的现价最低点）：
- 记录条件：`recording_price_low = true` 且 `strategy_active = true`
- 触发开始：检测到EMA12下穿EMA144时
- 触发停止：检测到第一次上穿时
- 记录对象：K线的最低价（`low`）
- 用途：与low1的平均值作为限价单价格

**上穿和下穿事件计数**:

#### crossover_count（EMA12上穿EMA144的次数）：
- 首次上穿（crossover_count = 1）：记录价格最低点low2
- 第二次上穿（crossover_count = 2）：触发限价单设置

#### crossunder_count（EMA12下穿EMA144的次数）：
- 首次下穿：启动price_low_after_cu的记录

## 4. 开仓逻辑

### 4.1 限价单设置条件

当满足以下条件时设置限价单：
1. `strategy_active = true`（策略已激活）
2. `crossover_count >= 2`（已检测到第二次上穿）
3. `ema12_low > 0`（已记录EMA12最低点）
4. `price_low_after_cu > 0`（已记录下穿后的现价最低点）

### 4.2 限价单价格计算

```
limit_price = (ema12_low + price_low_after_cu) / 2
```
- 使用EMA12最低点和下穿后现价最低点的平均值作为开仓价格
- 避免过于激进的开仓位置

### 4.3 开仓方式

- **订单类型**: 限价单
- **订单ID**: "Long1" 和 "Long2"
- **分批派单**: 两笔各50%仓位（每笔0.5单位）
- **仓位大小**: 总计1.0单位（账户权益的20%）

## 5. 止盈止损规则

### 5.1 多单止盈

**止盈条件**:
- 当收盘价 >= EMA144时，平仓一半仓位（0.5单位）
- 剩余0.5单位继续持有，由止损管理

**止盈特点**:
- EMA144作为中期关键阻力位
- 分批止盈可以锁定利润，同时保留趋势跟踪机会

### 5.2 多单止损

**止损价格**: `stop_loss_price = price_low_downtrend`
- 使用空头趋势中的现价最低点作为止损
- 保护下行风险

**止损订单**:
- 订单ID: "SL Long"
- 同时应用于 "Long1" 和 "Long2"
- 采用挂单止损方式

## 6. 状态转换流程

### 6.1 完整状态转换

```
初始状态（所有标记位为false）
        ↓
[1] 检测到多头趋势（EMA12 > EMA144 > EMA565）
    → was_uptrend = true
    → 清空: crossover_count, crossunder_count, ema12_low等所有数据
    → 状态显示: 蓝色方块 "1"
        ↓
[2] 检测到空头趋势（EMA12 < EMA144 < EMA565）且 was_uptrend = true
    → was_downtrend = true
    → strategy_active = true
    → 初始化: ema12_low = ema12, price_low_downtrend = close
    → 开始记录事件
    → 状态显示: 黄色方块 "2"
        ↓
[3] 在空头趋势中持续记录最低点和交叉事件
    → 记录 ema12_low（EMA12最低点）
    → 记录 price_low_downtrend（现价最低点）
    → 检测 crossover 和 crossunder 事件
    → 记录 price_low_after_cu（下穿后的现价最低点）
        ↓
[4] 当 crossover_count = 2（第二次上穿）
    → 计算 limit_price = (ema12_low + price_low_after_cu) / 2
    → 设置 limit_order_placed = true
    → 派出限价单
        ↓
[5] 限价单成交
    → 进入持仓状态
    → 执行止盈止损规则
        ↓
[6] 检测到多头趋势且 strategy_active = true
    → was_uptrend = true
    → was_downtrend = false
    → strategy_active = false
    → 清空计数器
    → 回到 [1] 状态
```

### 6.2 关键转换条件

**从多头到空头的转换必须满足**:
- 当前趋势：多头（`EMA12 > EMA144 > EMA565`）
- 状态标记：`was_uptrend = true`
- 新趋势：空头（`EMA12 < EMA144 < EMA565`）
- 结果：确认是从多头真正转变为空头，而不是中间的混乱状态

**事件只在策略激活时记录**:
- `strategy_active = true` 是所有事件记录的必要条件
- 防止在多头和空头之间的过渡期产生误信号

## 9. 策略特点

1. **精确的状态转换**:
   - 使用三个标记位确保状态清晰
   - 只有从多头转向空头时才激活策略
   - 防止在多空转换中产生错误信号

2. **关键最低点记录**:
   - 记录EMA12在空头趋势中的最低点
   - 记录现价在空头趋势中的最低点
   - 记录下穿后的现价最低点
   - 三个最低点相互配合确保精确开仓

3. **限价单开仓**:
   - 不是市价开仓，而是在特定价位下限价单
   - 提高开仓位置的精确度
   - 可以等待更优的入场点位

4. **分批派单**:
   - 将1单位的多单分为两笔各0.5单位
   - 更灵活的风险管理
   - 可独立平仓管理

5. **双阶段止盈**:
   - 第一阶段：在EMA144处平仓0.5单位，锁定利润
   - 第二阶段：剩余0.5单位由止损管理
   - 在获利和持仓之间找到平衡

6. **下行风险保护**:
   - 止损设置在空头趋势的现价最低点
   - 可以有效防止大幅下跌

## 10. 注意事项

1. **状态转换必须完整**:
   - 必须先进入多头趋势（was_uptrend = true）
   - 然后检测到空头趋势时才启动策略
   - 不会在一开始就是空头趋势时启动

2. **事件记录条件严格**:
   - 所有交叉事件和最低点记录都需要 `strategy_active = true`
   - 确保不会在多空转换的过渡期产生噪音信号

3. **限价单可能不成交**:
   - 如果价格没有跌回到限价单价格，将不会开仓
   - 此时需要等待下一个信号周期

4. **二次上穿触发**:
   - 需要等待EMA12第二次上穿EMA144才会设置限价单
   - 这样可以确保反转信号足够强烈

5. **止盈时机**:
   - 当价格达到EMA144时自动平仓0.5单位
   - EMA144作为动态阻力位，可根据市场调整

6. **仓位管理**:
   - 默认使用账户权益的20%作为仓位大小
   - 可在策略设置中调整

7. **回测注意**:
   - 在TradingView上测试时，确保图表时间框架为4小时
   - 确保有足够的历史数据（至少1年以上）
   - 注意点差和滑点的影响

## 11. 交易示例

### 完整的交易流程示例

**第1阶段：多头趋势**
- 图表显示：蓝色方块 "1"

- 所有计数器和记录值清零
- 等待趋势转换

**第2阶段：转为空头趋势**
- 图表显示：黄色方块 "2"

- 初始化：ema12_low, price_low_downtrend
- 开始监测交叉事件

**第3阶段：第一次下穿**
- 图表显示：红色向下三角 "下穿"
- crossunder_count = 1
- 启动 price_low_after_cu 的记录

**第4阶段：第一次上穿**
- 图表显示：绿色向上三角 "上穿"
- crossover_count = 1
- 停止 price_low_after_cu 的更新

**第5阶段：第二次上穿**
- 图表显示：绿色向上三角 "上穿"
- crossover_count = 2

- 如果价格跌破 price_low_downtrend：触发止损平仓
- 或者检测到多头趋势：清空所有状态，等待下一个信号
