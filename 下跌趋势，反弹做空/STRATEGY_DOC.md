# 4H Vegas Channel 下跌趋势策略文档（空单专用）

## 1. 策略概述

**策略名称**: 4H Vegas Channel Downtrend Strategy (Short Only)  
**时间周期**: 4小时  
**策略类型**: 下跌趋势中的反弹做空策略  
**交易方向**: 仅做空  
**交易理念**: 在空头趋势中，捕捉价格反弹至关键均线（EMA140）的做空机会  
**默认仓位**: 账户权益的20%

## 2. 技术指标

### 2.1 EMA指标
策略使用5条EMA均线：
- **EMA12**: 12周期指数移动平均线（黄色）
- **EMA140**: 140周期指数移动平均线（蓝色）
- **EMA156**: 156周期指数移动平均线（蓝色）
- **EMA565**: 565周期指数移动平均线（紫色）
- **EMA676**: 676周期指数移动平均线（紫色）

### 2.2 趋势判断

**下跌趋势信号 (downtrend_signal)**:
```
EMA12 < min(EMA140, EMA156) < min(EMA565, EMA676)
```
此条件确保价格处于明确的下跌趋势中，所有EMA均线按顺序排列。

### 2.3 斜率计算

**斜率函数**:
- 使用线性回归方法计算EMA140的28周期斜率
- 斜率单位：角度（度）
- 计算公式：使用最小二乘法计算斜率，然后转换为角度

**斜率条件**:
- `slope_above_25`: EMA140斜率 > 25度（用于空单信号准备）
- 斜率大于25度表示下跌趋势较为陡峭，适合做空
- **重要**: 信号准备时只需要"曾经"满足过斜率条件即可，一旦设置 `kong_signal_ready = true`，即使后续斜率变小，只要趋势还在，仍可等待入场机会

## 3. 交易逻辑

### 3.1 空单逻辑（反弹做空）

**策略思路**:
- 在空头趋势中，当价格快速向上反弹突破EMA140时，视为反弹做空机会
- 此时均线可能还未反应过来，但价格已经反弹到关键位置，适合做空

**信号准备条件**:
1. 当前未处于空单准备状态 (`kong_signal_ready == false`)
2. 当前满足下跌趋势信号 (`downtrend_signal == true`)
3. **曾经**满足EMA140斜率 > 25度 (`slope_above_25 == true`)
   - 注意：只需要在某个时刻满足过斜率条件即可，之后即使斜率变小，只要信号准备状态还在，仍可等待入场

**入场条件**:
1. 已设置空单准备信号 (`kong_signal_ready == true`)
2. **现价上穿EMA140** (`ta.crossover(close, ema140)`)
   - 价格从下方突破到EMA140上方，这是反弹信号
3. 当前无持仓 (`strategy.position_size == 0`)

**入场操作**:
- 入场价格：当前收盘价（价格突破EMA140时的价格）
- 仓位大小：分两笔入场，每笔0.5个单位（总计1个单位）
- 订单ID：`"Short1"` 和 `"Short2"`

**信号重置机制**:
1. **开单后立即重置**: 开单成功后，立即设置 `kong_signal_ready = false`，确保该信号只触发一次
2. **趋势消失时重置**: 当不再满足下跌趋势信号时，重置 `kong_signal_ready = false`
3. **重新准备**: 只有重新同时满足以下条件，才能再次设置 `kong_signal_ready = true`：
   - 当前满足下跌趋势信号 (`downtrend_signal == true`)
   - 当前满足EMA140斜率 > 25度 (`slope_above_25 == true`)

## 4. 止盈止损规则

### 4.1 空单止盈止损

**止盈规则**（按优先级，满足即平仓）:
1. 价格达到入场价的 **75%** (`close <= entry_price * 0.75`) - 最大止盈目标
2. 价格达到入场价的 **85%** (`close <= entry_price * 0.85`) - 中等止盈目标
3. 价格达到入场价的 **90%** (`close <= entry_price * 0.90`) - 最小止盈目标

**止损规则**:
- 止损价格：入场价的 **105%** (`entry_price * 1.05`)
- 止损订单ID：`"SL Short"`（同时应用于Short1和Short2）
- 风险回报比：最大约1:5（止损5% vs 最大止盈25%）

## 5. 移动止损机制

### 5.1 价格记录

**记录条件**:
- 仅在持仓期间记录价格
- 如果收盘价 > EMA12：记录到 `highprice` 数组
- 如果收盘价 <= EMA12：记录到 `lowprice` 数组

**数组初始化**:
- 开仓时清空价格数组
- 平仓后清空价格数组

### 5.2 移动止损计算

**启用条件**:
- 价格记录总数 > 8条 (`total_count > 8`)

**空单移动止损逻辑**:
- 如果 `highprice数量 > lowprice数量`：
  - 移动止损 = 入场价 (`trailing_stop = entry_price`)
  - 说明：价格上涨次数多于下跌次数，保持原止损
- 否则：
  - 移动止损 = `highprice数组的平均值`
  - 说明：价格下跌趋势良好，将止损调整至高价平均值，锁定利润

**注意**: 当前代码中计算了移动止损值，但未实际应用到止损订单中。

## 8. 策略特点

1. **反弹做空策略**:
   - 在空头趋势中，捕捉价格反弹至EMA140的做空机会
   - 价格快速反弹时，均线可能还未反应过来，此时是较好的做空时机
   - 通过斜率过滤确保趋势足够明确

2. **斜率过滤**:
   - 通过EMA140斜率（>25度）确保下跌趋势足够陡峭
   - 避免在平缓或震荡市场中交易

3. **分批入场**:
   - 空单采用分批入场（两笔各0.5单位），用于风险控制
   - 可以灵活管理仓位

4. **多级止盈**:
   - 采用三个止盈价位（90%、85%、75%），逐步锁定利润
   - 最大潜在收益25%，最小10%

5. **移动止损机制**:
   - 虽然计算了移动止损，但当前未实际应用到止损订单中
   - 可根据价格走势动态调整止损位置

## 9. 注意事项

1. **移动止损未启用**: 代码中计算了移动止损值，但未实际应用到止损订单中。如需启用，需要修改止损订单逻辑。

2. **空单分批入场**: 空单分两笔入场（Short1和Short2），但止盈时同时平仓两笔。

3. **信号重置机制**: 
   - 开单后立即重置信号，确保每个准备信号只触发一次
   - 当趋势信号消失时（不再满足下跌趋势条件），会重置空单准备信号
   - 只有重新满足空头趋势和斜率条件后，才能再次准备新的信号

4. **仓位管理**: 默认使用账户权益的20%作为仓位大小，可通过策略设置调整。

5. **入场时机**: 在空头趋势中，当价格向上反弹突破EMA140时入场做空。这是反弹做空策略，利用价格反弹至关键阻力位的机会。

6. **风险控制**: 止损设置为5%，最大止盈为25%，风险回报比约为1:5。
