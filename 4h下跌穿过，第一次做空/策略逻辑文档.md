# 4H下跌穿过第一次做空策略 - 逻辑文档

## 一、策略概述

本策略旨在捕捉多头趋势中的反弹高点，在EMA12下穿EMA565后，等待反弹高点确认，然后在高点下方开空单。

**核心逻辑**: 
- **触发条件**: 曾经处于多头趋势，当第一次发生下穿时才执行策略
- **执行时机**: 不需要一直保持多头趋势，只要曾经是多头趋势，然后第一次下穿EMA565时，就开始执行策略
- **下穿后状态**: 下穿后趋势已经不是多头了（`ema12 < ema565 < ema144`），但策略继续执行
- **记录期间**: 只需要不是空头趋势即可，不需要保持多头趋势

## 二、完整流程

### 阶段1: 趋势识别与下穿检测

#### 1.1 记录曾经处于多头趋势的状态
- **条件**: `ema565 > ema144 > ema12`
- **状态**: `uptrend_state = 1`（记录曾经处于多头趋势）
- **说明**: 
  - 当满足多头趋势条件时，记录"曾经处于多头趋势"的状态
  - 这个状态用于后续的下穿检测
  - **不需要一直保持多头趋势**，只要曾经是多头趋势即可
  - **可以重复触发**: 即使策略运行中（`uptrend_state == 2`或已开仓），如果重新出现多头趋势，可以重新记录状态

#### 1.2 第一次下穿检测（策略触发）
- **触发条件**: 
  - **曾经处于多头趋势**（`uptrend_state == 1`）- **这是必要条件**
  - EMA12**第一次**下穿EMA565（`ta.crossunder(ema12, ema565)`）
  - 下穿后状态：`ema12 < ema565 < ema144`（**此时已经不是多头趋势了**）
- **状态转换**: `uptrend_state = 2`（开始执行策略，记录最高价）
- **初始化变量**:
  - `highest_price_since_crossdown = high`（记录当前最高价）
  - `bars_since_highest = 0`（重置最高价后的K线计数）
- **重要说明**: 
  - **策略触发时机**: 曾经处于多头趋势 + 第一次下穿EMA565
  - 下穿后，趋势已经不是多头（`ema12 < ema565`）
  - 但也不是空头（`ema565 < ema144`）
  - 这是一个过渡状态，但策略继续执行
- **策略可重复触发**:
  - **即使策略运行中**（`uptrend_state == 2`或已开仓，止损还没出现），如果重新出现多头趋势，并且是多头趋势后的第一次下穿EMA565，**可以再次触发策略**
  - 这意味着可以同时有多个策略实例运行，或者重置当前策略重新开始

#### 1.3 状态重置与重复触发
- **重置为0**: 如果形成空头趋势（`ema12 < ema144 < ema565`）
  - 一旦变成空头趋势，所有记录清零，重新等待
  - 记录期间只需要不是空头趋势即可，不需要保持多头趋势
- **策略重复触发**:
  - **场景**: 当策略运行中（`uptrend_state == 2`或已开仓），止损还没出现
  - **条件**: 重新出现多头趋势（`ema565 > ema144 > ema12`），然后第一次下穿EMA565
  - **行为**: 可以再次触发策略
  - **说明**: 
    - 可以重置当前策略状态，重新开始记录最高价
    - 或者可以同时运行多个策略实例（取决于实现方式）
    - 这允许策略在多个周期中重复执行

### 阶段2: 最高价记录与开仓条件

**重要说明**: 
- 此阶段在 `uptrend_state == 2`（已下穿状态）下运行
- **不需要保持多头趋势**，下穿后趋势已经不是多头了
- 只需要**不是空头趋势**即可继续记录
- 如果变成空头趋势（`ema12 < ema144 < ema565`），则重置状态

#### 2.1 最高价记录逻辑 ✅ 已明确
- **记录时机**: `uptrend_state == 2`（已下穿状态）
- **记录条件**: 不是空头趋势（`not is_downtrend`）
- **记录方式**: 
  - **记录K线的最高价**（`high`）- ✅ 已明确
  - 每根K线检查 `high`（K线最高价）
  - 如果 `high > highest_price_since_crossdown`，更新最高价并重置 `bars_since_highest = 0`
  - 否则 `bars_since_highest += 1`（最高价未更新的K线数+1）
- **说明**: 记录的是每根K线的最高价（high），不是收盘价（close）

#### 2.2 开仓条件 ✅ 已明确
- **条件**: 
  - `bars_since_highest >= 42`（最高价连续42根K线未更新）
  - `na(entry_price)`（未开仓）
  - `strategy.position_size == 0`（无持仓）
- **开仓位置**: 
  - 价格1: `最高价 * (1 - 0.01)` = 低于最高价1%
  - 价格2: `最高价 * (1 - 0.02)` = 低于最高价2%
- **开仓方式**: 限价单（limit order）
- **订单管理**: ✅ **已明确** - 订单最好分别管理，或者用买入多少管理
  - **方式1**: 分别管理两个订单（Short_1和Short_2），分别记录成交价格和持仓
  - **方式2**: 用买入数量管理，记录总持仓和平均成本
- **说明**: 认为42根K线最高价未更新，说明反弹高点已确认

### 阶段3: 开仓后的管理逻辑

#### 3.1 开仓成交检测 ✅ 已明确
- **检测条件**: `na(entry_price) and strategy.position_size < 0`
- **记录信息**:
  - **订单分别管理方式**:
    - `entry_price_1` = 价格1的成交价格（如果Short_1成交）
    - `entry_price_2` = 价格2的成交价格（如果Short_2成交）
    - `entry_qty_1` = Short_1的持仓数量
    - `entry_qty_2` = Short_2的持仓数量
  - **或者用买入数量管理方式**:
    - `entry_price` = 平均成本价（根据成交价格和数量计算）
    - `total_position_size` = 总持仓数量
  - `entry_highest_price = high`（记录开仓时的最高价）
  - `bars_since_entry = 0`（重置开仓后K线计数）
  - `close_above_entry_count = 0`（重置收盘价大于开仓价计数）
- **说明**: 
  - 订单需要分别管理，或者用买入数量管理
  - 如果两个订单都成交，需要分别记录各自的成交价格

#### 3.2 站稳判断与平仓逻辑（逻辑1）✅ 已明确
- **记录方式**: 
  - **当K线的收盘价高于成本（开仓价）时，计数器+1** - ✅ 已明确
  - 每根K线检查 `close > entry_price`，如果是则 `close_above_entry_count += 1`
  - **累计计数**：每次收盘价高于成本就计数+1，不重置
- **站稳定义**: `close_above_entry_count > 4`（收盘价大于开仓价次数 > 4）
- **平仓条件1（成本平仓）**: 
  - 已站稳（`close_above_entry_count > 4`）
  - 且当前价格低于开仓成本（`close < entry_price`）
- **平仓动作**: `strategy.close_all(comment="站稳后低于成本平仓")`
- **说明**: 
  - 如果价格多次站上开仓价，说明可能反转，一旦跌破成本就平仓
  - 计数是累计的，每次收盘价高于成本就+1，不重置

#### 3.3 8根K线内站稳现价平仓（逻辑2）
- **条件**: 
  - `bars_since_entry <= 8`（8根K线内）
  - 已站稳（`close_above_entry_count > 4`）
  - 但价格未低于成本（`close >= entry_price`）
- **平仓动作**: `strategy.close_all(comment="8根K线内站稳现价平仓")`
- **说明**: 8根K线内如果站稳了，但价格一直没低于成本，直接现价平仓

#### 3.4 未站稳继续持有（逻辑3）
- **条件**: 
  - 未站稳（`close_above_entry_count <= 4`）
  - 持仓中（`strategy.position_size < 0`）
- **行为**: 
  - **一直持有仓位**，直到触发止盈条件
  - 不设置强制平仓时间限制
  - 可以设置止损作为保护（可选）
- **止损价格（可选）**: `ta.highest(high, 8)`（最近8根K线的最高价）
- **动作**: `strategy.exit("SL_8bars", stop=highest_in_8_bars)`（如果设置止损）
- **说明**: 
  - **核心逻辑**: 如果没站稳，说明趋势可能继续，仓位应该一直拿着
  - **平仓方式**: 主要通过止盈逻辑（3.5）平仓
  - **止损作用**: 作为保护性止损，防止意外情况

#### 3.5 止盈逻辑
- **触发条件**: 出现空头趋势（`is_downtrend`）且持仓中
- **止盈位置**:
  - TP1: `entry_price * (1 - 0.10)` = 低于开仓价10%
  - TP2: `entry_price * (1 - 0.20)` = 低于开仓价20%
  - TP3: `entry_price * (1 - 0.26)` = 低于开仓价26%
- **执行顺序**: 按TP3 → TP2 → TP1的顺序检查

- 步骤1：记录"曾经处于多头趋势"的状态（不需要一直保持，可以重复记录）
- 步骤2：曾经处于多头趋势 + 第一次下穿 → 触发策略执行（可以重复触发）
- 步骤3：下穿后趋势已经不是多头，但策略继续执行
- 步骤4：记录期间不需要保持多头趋势，只需要不是空头趋势
- **步骤8（重要）**: 策略可以重复触发，即使之前的策略还在运行中

## 七、策略重复触发机制

### 7.1 重复触发的条件

**场景**: 当策略运行中（`uptrend_state == 2`或已开仓），止损还没出现

**触发条件**:
1. 重新出现多头趋势（`ema565 > ema144 > ema12`）
2. 并且是多头趋势后的第一次下穿EMA565（`ta.crossunder(ema12, ema565)`）

**行为**:
- 可以再次触发策略
- 重置当前策略状态（`uptrend_state = 2`）
- 重新初始化变量（`highest_price_since_crossdown`、`bars_since_highest`等）
- 重新开始记录最高价

### 7.2 重复触发的影响

**对已开仓的影响**:
- 如果已经开仓（`strategy.position_size < 0`），重复触发会重置记录状态
- **需要明确**: 是否保留已开仓的持仓？还是重置所有状态包括持仓？
- **建议**: 保留已开仓的持仓，只重置记录状态，允许新的策略实例运行

**对记录状态的影响**:
- 重置 `highest_price_since_crossdown` 和 `bars_since_highest`
- 重新开始记录最高价
- 如果之前已经挂限价单但未成交，需要取消旧订单

### 7.3 实现方式建议

**方式1: 重置当前策略状态**
- 重置记录状态，保留已开仓的持仓
- 允许新的策略实例与旧的持仓同时存在
- 新的策略实例独立记录和开仓

**方式2: 完全重置**
- 重置所有状态，包括已开仓的持仓
- 平掉所有持仓，重新开始
- 简单但可能丢失已有持仓

**方式3: 多实例管理**
- 允许同时运行多个策略实例
- 每个实例独立管理自己的状态和持仓
- 复杂但灵活

## 九、逻辑拆分说明

### 站稳判断的计数规则 ✅ 已明确

**计数方式**:
- **触发条件**: 当K线的收盘价高于成本（开仓价）时，计数器+1
- **计数规则**: `if close > entry_price then close_above_entry_count += 1`
- **累计计数**: 每次收盘价高于成本就计数+1，**不重置**
- **站稳定义**: `close_above_entry_count > 4`（累计次数 > 4）
- **说明**: 
  - 即使价格回落到成本以下，计数也不重置
  - 只要累计次数 > 4，就认为站稳了
  - 计数是累计的，不是连续的

### 站稳判断的两种平仓方式

#### 情况1: 站稳 + 价格低于成本
- **条件**: `close_above_entry_count > 4` 且 `close < entry_price`
- **动作**: 成本平仓（逻辑1）
- **说明**: 站稳后价格回落，跌破成本时平仓

#### 情况2: 站稳 + 价格未低于成本（8根K线内）
- **条件**: `bars_since_entry <= 8` 且 `close_above_entry_count > 4` 且 `close >= entry_price`
- **动作**: 现价平仓（逻辑2）
- **说明**: 8根K线内站稳了，但价格一直没低于成本，直接现价平仓

#### 情况3: 未站稳（一直持有）
- **条件**: `close_above_entry_count <= 4`（未站稳）
- **动作**: 一直持有仓位，直到触发止盈条件（逻辑3）
- **说明**: 
  - **核心逻辑**: 如果没站稳，说明趋势可能继续，仓位应该一直拿着
  - **平仓方式**: 主要通过止盈逻辑平仓（出现空头趋势时，在10%、20%、26%止盈）
  - **止损作用**: 可以设置止损作为保护（可选），但不强制平仓
  - **无时间限制**: 不设置8根K线强制平仓，一直持有直到止盈
